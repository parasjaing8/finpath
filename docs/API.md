# API Reference — ai-chat Platform

> Base URL: `http://192.168.0.130:8080`
> Last updated: 2026-04-14

---

## WebSocket

### `WS /ws`
Main bidirectional channel for all chat and orchestration activity.

**Connection:** `ws://192.168.0.130:8080/ws`

On connect, the server immediately sends:
```json
{"type": "history", "messages": [...]}
{"type": "status",  "claude_online": true}
```

---

### Client → Server Messages

| `type` | Fields | Description |
|--------|--------|-------------|
| `content` | `content: str`, `project_id?: int` | User chat message. If `project_id` is present, routed as project-scoped message. |
| `start_orchestration` | `project_id: int`, `goal: str` | Begin full project build. |
| `resume_orchestration` | `project_id: int` | Resume interrupted build (resumes pending tasks). |
| `fix_project` | `project_id: int`, `feedback: str` | Targeted fix based on user feedback. |
| `get_projects` | — | Request current project list. |
| `load_project` | `project_id: int` | Load project messages, tasks, and state. |
| `cancel` | — | Cancel the currently running task/orchestration. |

#### `/exec` shortcut
Send a chat message starting with `/exec <cmd_key>` to run a whitelisted server command.
Available keys: `ollama-list`, `ollama-ps`, `disk-usage`, `uptime`, `server-log`, `server-status`, `memory`.

---

### Server → Client Messages

| `type` | Fields | Description |
|--------|--------|-------------|
| `history` | `messages: []` | Initial message history on connect. |
| `status` | `claude_online: bool` | Claude API availability. |
| `chunk` | `agent: str`, `content: str` | Streaming token delta from an agent. |
| `done` | `agent: str` | Agent has finished streaming. |
| `user` | `content: str`, `timestamp: str` | Echo of user's message. |
| `intent_detected` | `intent: str`, `original_message: str`, `name?: str`, `project_id?: int` | Claude detected project intent; UI must confirm. Intent: `project_new` or `project_continue`. |
| `plan` | `tasks: []` | Task list generated by Claude for a project. |
| `task_start` | `task_id: int`, `title: str` | A task is beginning execution. |
| `task_update` | `task_id: int`, `status: str` | Task status changed. |
| `task_complete` | `task_id: int` | Task completed successfully. |
| `project_list` | `projects: []` | Response to `get_projects`. |
| `project_loaded` | `project: {}`, `messages: []`, `tasks: []` | Response to `load_project`. |
| `complete` | `summary: str`, `play_url: str` | Project build finished. |
| `cancelled` | — | Active task was cancelled. |
| `error` | `message: str` | An error occurred during processing. |
| `agent_count` | `count: int` | Number of agents responding (before their chunks arrive). |

## REST Endpoints

### General

#### `GET /`
Serves the chat UI (`static/index.html`).

#### `GET /config`
```json
{"server_host": "192.168.0.130:8080"}
```

#### `GET /health`
```json
{"ok": true, "uptime_s": 3600.5, "db": "ok"}
```

#### `GET /status`
```json
{"claude_online": true}
```

#### `GET /stats`
System resource stats (CPU, RAM, swap, network, Ollama status).
```json
{
  "cpu": 12.3,
  "ram_used": 8.1, "ram_total": 16.0,
  "swap_used": 0.0, "swap_total": 0.0,
  "rx_kbs": 45.2,  "tx_kbs": 12.1,
  "ollama_online": true
}
```

### Projects

#### `GET /projects`
List all projects.
```json
[{"id": 1, "name": "Snake Game", "slug": "snake-game", "status": "completed", ...}]
```

#### `POST /projects`
Create a new project.
```json
// Request
{"name": "My App", "description": "A click counter"}

// Response
{"id": 2, "slug": "my-app", "folder_path": "...", "status": "active", ...}
```

#### `GET /projects/{id}`
Get a single project by ID.

#### `DELETE /projects/{id}`
Delete project and all its messages, tasks, and source files.
```json
{"ok": true}
```

#### `GET /projects/{id}/files`
List all source files in `projects/<slug>/src/`.
```json
[{"path": "index.html", "size": 4096}, {"path": "js/game.js", "size": 12800}]
```

#### `GET /projects/{id}/files/{file_path}`
Read a specific source file's content.
```json
{"path": "js/game.js", "content": "..."}
```

### Settings

#### `GET /settings`
Full agent configuration (models, enabled status, API key preview).

#### `POST /settings/agent`
Enable or disable an agent.
```json
// Request
{"agent": "qwen35", "enabled": false}

// Response
{"agent": "qwen35", "enabled": false, "disabled_agents": ["qwen35"]}
```

#### `POST /settings/apikey` *(localhost only)*
Save a new Anthropic API key to `.env`.
```json
// Request
{"key": "sk-ant-..."}

// Response
{"ok": true, "error": null}
```

#### `GET /settings/test/{agent}`
Test whether an agent is reachable.
```json
{"online": true}
```

#### `GET /settings/master`
Get current master model configuration.
```json
{"master_model": "claude", "claude_enabled": true, "effective_master": "claude"}
```

#### `POST /settings/master`
Change the master model.
```json
// Request
{"model": "claude", "claude_enabled": true}
```

#### `GET /settings/coder-model`
Get the current coder model (deepseek slot).

#### `POST /settings/coder-model`
Swap the coder model at runtime (used by benchmarks).
```json
// Request
{"model": "qwen3.5:7b", "label": "Qwen 7B"}
```

### Skills

#### `GET /skills`

#### `GET /skills/{filename}`

```json
// Request
{"content": "---\nname: ...\n---\nSkill body..."}
```

### Ollama Model Registry

#### `GET /ollama/models`
List all available Ollama models and their registration status.
```json
[{"name": "deepseek-coder-v2:16b...", "size_gb": 11.2, "registered_as": "deepseek", "builtin": true, "offline": false}]
```

#### `POST /ollama/models`
Register a new Ollama model as an agent.
```json
// Request
{"key": "qwen7b", "model": "qwen:7b", "label": "Qwen 7B"}

// Response
{"ok": true, "key": "qwen7b", "model": "qwen:7b", "label": "Qwen 7B"}
```

#### `DELETE /ollama/models/{key}`
Remove a custom agent registration. Built-in agents cannot be removed.

#### `DELETE /ollama/uninstall/{model_name}`
Delete a model from Ollama disk (`ollama rm`). Model must not be in use.

---

### Admin *(localhost only)*

#### `GET /admin/exec/commands`
List available exec command keys.

#### `POST /admin/exec`
Run a whitelisted server command.
```json
// Request
{"cmd": "disk-usage"}

// Response
{"ok": true, "cmd": "disk-usage", "stdout": "Filesystem ...", "returncode": 0}
```

#### `POST /backup`
Trigger an immediate backup of `chat.db` and `memory/` to `backups/<timestamp>/`.

---

### Static File Serving

#### `GET /static/{path}`
Serves files from `static/` directory (CSS, JS).

#### `GET /play/{slug}/`
Serves `projects/<slug>/src/index.html` (auto-generates if missing).

#### `GET /play/{slug}/{file_path}`
Serves any file from `projects/<slug>/src/`. Returns 403 for path traversal attempts.
